Multi-ISP Anycast Web Performance Monitor

Automated Zabbix 7.4 Solution for Tracking Shifting CDN Endpoints Across Dual Transit Providers

How It Works

When monitoring modern enterprise web platforms behind content delivery networks (CDNs) or Anycast groups, a domain name does not point to a fixed server. Instead, DNS resolutions return completely different IP addresses over time, sometimes rotating every minute. Standard monitoring architectures fall short here, as they evaluate the website as a single endpoint, obscuring path-specific performance anomalies.

This open-source solution framework leverages **Zabbix Low-Level Discovery (LLD)** to perform continuous target discovery and diagnostic triage. A master external script dynamically orchestrates simultaneous lookups over independent WAN links—binding to specific source interfaces—allowing you to inspect hop paths, latency variations, and transactional status matrices concurrently per active IP endpoint.

The Technical Core Stack

This project achieves highly granular, automated path tracing by leveraging standard Linux networking utilities coordinated through native Zabbix 7.4 configuration patterns:

Zabbix 7.4 LLD Engine

Acts as the state tracking mechanism. It interprets structured JSON data payloads from external scripts, dynamically creating and destroying dependent item prototypes and alarm triggers on the fly.

Python 3 Execution Engine

Leverages asynchronous lightweight worker threads to execute multi-tenant network diagnostic queries quickly, ensuring that slow traceroute actions do not serialize or choke system processes.

MTR (My Traceroute)

Combines classical ping and traceroute functionality. It queries intermediate layer-3 hops along transit paths, printing autonomous system numbers (ASN) directly inside diagnostic logs.
