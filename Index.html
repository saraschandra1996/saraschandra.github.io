Multi-ISP Anycast Web Performance Monitor

Automated Zabbix 7.4 Solution for Tracking Dynamic CDN Endpoints across Dual Providers

Overview

When monitoring high-availability websites behind Anycast networks or CDNs (Cloudflare, AWS CloudFront), target IP addresses rotate frequently. This solution uses Zabbix Low-Level Discovery (LLD) to discover active IP endpoints dynamically and monitor latency, operational availability, and hop paths (MTR) independently across a Primary and Backup ISP.

Features

  • Dynamic Variable Matching: Tracks multi-ISP performance parameters simultaneously.
  • Zero Stale Alerts: Uses Zabbix 7.4 native nodata() logic constraints to instantly silence alerts on retired Anycast IPs.

1. The Script Framework (check_web_site.py)

Deploy this file to your Zabbix server external scripts directory: /usr/lib/zabbix/externalscripts/. Ensure it has execution permissions (chmod +x).

# The script must return a clean JSON payload mapping the discovered properties:
[
  {
    "{#AWSIP}": "172.217.171.110",
    "isp1_name": "JIO",
    "isp2_name": "Pionner",
    "isp1_status": 1,
    "isp1_latency": 15.4,
    "isp1_trace": "1. AS... \n2. AS..."
  }
]

2. Zabbix Template Configuration

Import the Web Performance Monitoring-Dual-ISP.yaml configuration file included in this repository into your Zabbix 7.4 instance. It configures the following safety-gated expression rules:

# Primary Path Alert Expression
(max(/Web Performance Monitoring-Dual-ISP/net.multi_isp.latency.primary[{#AWSIP}],10m)=0 
or min(/Web Performance Monitoring-Dual-ISP/net.multi_isp.latency.primary[{#AWSIP}],3m)>300)
and nodata(/Web Performance Monitoring-Dual-ISP/net.multi_isp.status.primary[{#AWSIP}],15m)=0

Note: If the script drops an old IP, data collection stops. After 15 minutes, the nodata()=0 boundary drops to False, smoothly disabling the trigger to prevent any false alerts while keeping historical metrics intact.

3. Grafana Integration Dashboard

To avoid empty panels during rapid Anycast rotations, ensure you use the following transformation workflow:

  1. Set your query type mode explicitly to Text.
  2. Add the Labels to fields transformation and set the mode context to Rows.
  3. Run an Extract fields (JSON) pass to dynamically split the payload data frame streams into real-time metrics.
