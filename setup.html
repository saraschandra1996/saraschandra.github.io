System Deployment Guide

Prerequisites, Script Installation, Interface Binding, and Template Ingestion Steps

System Prerequisites

Before installing the Zabbix configurations, the proxy or server executing the external checks must have the necessary system packages and network capabilities installed:

  • Python 3.6+ with standard subprocess, threading, and json modules.
  • MTR Binary: Must support autonomous system inspection flags (-z / --aslookup).
  • Linux Capabilities: The user process running Zabbix (typically zabbix) requires permissions to open raw sockets for ping and interface binding operations.

Execute the following commands on your Zabbix worker node to install utilities and assign network capability flags:

# Install MTR on Debian/Ubuntu systems
sudo apt-get update && sudo apt-get install mtr-tiny -y

# Grant Zabbix components permission to issue raw packet binding commands
sudo setcap cap_net_raw+ep /bin/ping
sudo setcap cap_net_raw+ep /usr/bin/mtr-raw

1 Install External Scripts

Drop check_web_site.py and isp_config.py directly into your system's designated Zabbix external scripts directory, which defaults to /usr/lib/zabbix/externalscripts/.

Open isp_config.py and modify the source parameters to match your localized infrastructure constraints:

# File destination: /usr/lib/zabbix/externalscripts/isp_config.py
ISP1_SOURCE_IP = "203.0.113.10"   # The source interface IP of your Primary ISP (e.g., JIO)
ISP1_NAME = "JIO"

ISP2_SOURCE_IP = "198.51.100.25"  # The source interface IP of your Backup ISP (e.g., Pionner)
ISP2_NAME = "Pionner"

Set correct ownership and executable flags so the Zabbix daemon can spawn the child worker processes cleanly:

sudo chown zabbix:zabbix /usr/lib/zabbix/externalscripts/check_web_site.py
sudo chown zabbix:zabbix /usr/lib/zabbix/externalscripts/isp_config.py
sudo chmod +x /usr/lib/zabbix/externalscripts/check_web_site.py

2 Import Zabbix Template

Download the dual_isp_template.yaml file from the repository root. Navigate to your Zabbix 7.4 Web Interface and follow these configuration steps:

  1. Go to Data collection ➡️ Templates and click Import in the top right corner.
  2. Choose the dual_isp_template.yaml file.
  3. Keep the rules at their defaults (Update existing, Create new) and click Import.
Key Design Principle: The template contains an absolute Master External Item configured with an operational interval of 30s. The underlying Discovery Rule runs as a Dependent Item mapping directly to this Master. Never configure the LLD rule to cycle faster than the Master Item interval, as this will result in data processing gaps and item flapping.

3 Assign Host Macro Constants

Once imported, link the newly created template to your target host configuration block:

  1. Navigate to Data collection ➡️ Hosts and select your monitoring asset.
  2. Click on the Macros tab and select Inherited and host macros.
  3. Locate the {$AWS_SITE_URL} context row and assign your target website address (e.g., go.proctoru.com). Click Update.
